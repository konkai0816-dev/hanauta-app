# 要件定義書

## 1. プロジェクト概要

### 1.1 目的
鼻歌を録音するだけで曲名・アーティスト名を特定できるWebアプリを提供する。
「あの曲なんだっけ？」という場面で手軽に使えるツールを目指す。

### 1.2 対象ユーザー
- 一般公開（不特定多数）
- 同時利用人数：10人未満を想定

---

## 2. 機能要件

### 2.1 必須機能（MVP）

| # | 機能 | 詳細 |
|---|------|------|
| F-01 | 鼻歌録音 | ブラウザ上のボタンを押して録音開始・停止 |
| F-02 | 録音時間制限 | 10秒〜30秒の範囲 |
| F-03 | 曲認識 | 録音した音声をACRCloud APIに送信して解析 |
| F-04 | 結果表示 | 上位3曲を信頼度（%）付きで表示 |
| F-05 | 注意事項表示 | 「日本語楽曲のみ対応」の旨をUI上に明記 |
| F-06 | 再録音 | 結果表示後、もう一度録音できる |

### 2.2 結果表示の仕様
- 1位〜3位の候補曲を信頼度の高い順に表示
- 各候補に表示する情報：
  - 曲名
  - アーティスト名
  - アルバム名（取得できる場合）
  - 信頼度（%）
- 認識できなかった場合は「曲を特定できませんでした」と表示

### 2.3 対応楽曲
- 日本語楽曲のみ（J-POP, アニソン, 演歌 等）
- 洋楽・インスト曲は対象外（認識できない場合がある旨を明記）

---

## 3. 非機能要件

| 項目 | 内容 |
|------|------|
| レスポンス | 認識結果は30秒以内に返す |
| 同時接続 | 10人未満（順番待ちの仕組みは不要） |
| ブラウザ対応 | Chrome最新版を必須対応とする |
| セキュリティ | APIキーはサーバーサイドのみで管理、フロントに公開しない |
| コスト | ACRCloud無料枠（1,000回/日）の範囲内で運用 |

---

## 4. 画面構成

### 4.1 メイン画面（1画面構成）

```
+------------------------------------------+
|        鼻歌で曲を当てよう！               |
|  ※日本語楽曲のみ対応しています           |
+------------------------------------------+
|                                          |
|   [録音開始ボタン]                        |
|   録音時間: 0秒 / 最大30秒               |
|                                          |
|   --- 認識中スピナー（処理中のみ表示）--- |
|                                          |
|   結果:                                  |
|   1位: 曲名 / アーティスト  90%          |
|   2位: 曲名 / アーティスト  75%          |
|   3位: 曲名 / アーティスト  60%          |
|                                          |
|   [もう一度試す]                          |
+------------------------------------------+
```

---

## 5. 制約・前提条件

- マイクのアクセス許可がブラウザで必要
- HTTPS環境（またはlocalhost）でのみマイクが使える
- ACRCloud APIの利用にはアカウント登録・APIキー取得が必要
